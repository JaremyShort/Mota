# 魔塔游戏需求文档

## 1. 项目背景与游戏目标

### 1.1 项目背景
经典魔塔是一款像素风格的RPG解谜游戏，以其独特的数值策略玩法著称。玩家需要在50层的塔楼中探索，通过合理的资源分配和战斗策略，逐步提升实力，最终击败顶层魔王。

### 1.2 游戏目标
- **主线目标**：击败50层的魔王，解救被困的公主
- **核心体验**：数值策略 + 解谜探索 + 资源管理
- **胜利条件**：到达50层并击败最终BOSS

### 1.3 目标用户
- 喜欢像素风RPG的玩家
- 热爱策略解谜的游戏爱好者
- 追求怀旧经典游戏体验的用户

## 3. 游戏数值设计要点

### 3.1 战斗公式

```
勇士对怪物伤害 = max(1, 勇士攻击 - 怪物防御)
怪物对勇士伤害 = max(1, 怪物攻击 - 勇士防御)

战斗回合数 = ceil(怪物HP / 勇士对怪物伤害)
勇士总损血 = 战斗回合数 × 怪物对勇士伤害
```

### 3.2 属性设计原则

| 属性 | 设计要点 |
|-----|---------|
| 生命值(HP) | 初始1000，上限无限制，通过血瓶和升级增加 |
| 攻击力(ATK) | 初始10，通过武器和宝石提升 |
| 防御力(DEF) | 初始10，通过防具和宝石提升 |
| 金币(GOLD) | 击杀怪物获得，用于商店购买 |
| 经验值(EXP) | 击杀怪物获得，累计升级 |

### 3.3 怪物属性分布

| 楼层范围 | 怪物特点 | 典型HP范围 | 典型ATK范围 | 典型DEF范围 |
|---------|---------|-----------|-----------|-----------|
| 1-10层 | 新手区 | 50-500 | 20-80 | 10-30 |
| 11-20层 | 初级区 | 500-2000 | 80-200 | 30-80 |
| 21-30层 | 中级区 | 2000-8000 | 200-500 | 80-200 |
| 31-40层 | 高级区 | 8000-30000 | 500-1500 | 200-600 |
| 41-50层 | Boss区 | 30000-100000 | 1500-5000 | 600-2000 |

### 3.4 道具数值设计

| 道具类型 | 效果 | 分布楼层 |
|---------|-----|---------|
| 小血瓶 | +200 HP | 1-20层 |
| 大血瓶 | +500 HP | 11-40层 |
| 超级血瓶 | +1000 HP | 21-50层 |
| 红宝石 | +2 ATK | 全塔 |
| 蓝宝石 | +2 DEF | 全塔 |
| 钥匙 | 开启对应门 | 全塔 |

## 4. 技术栈建议

### 4.1 推荐方案：HTML5 Canvas + 原生JavaScript

```
技术栈组成：
├── HTML5 Canvas    # 游戏渲染引擎
├── JavaScript ES6+ # 游戏逻辑
├── CSS3            # UI样式
└── localStorage    # 数据持久化
```

### 4.2 技术选型理由

| 方面 | 选择理由 |
|-----|---------|
| **性能** | Canvas直接操作像素，渲染效率高，适合图块游戏 |
| **开发效率** | 原生JS无框架开销，代码直观，易于调试 |
| **兼容性** | 所有现代浏览器支持，无需构建工具 |
| **学习成本** | 技术栈简单，适合快速原型开发 |
| **部署简单** | 静态文件即可运行，无需服务器 |

### 4.3 项目结构建议

```
web_mota/
├── index.html          # 游戏入口
├── css/
│   └── style.css       # 样式文件
├── js/
│   ├── main.js         # 主程序入口
│   ├── game.js         # 游戏核心逻辑
│   ├── map.js          # 地图管理
│   ├── player.js       # 玩家控制
│   ├── monster.js      # 怪物系统
│   ├── battle.js       # 战斗系统
│   ├── item.js         # 道具系统
│   ├── ui.js           # 界面渲染
│   └── save.js         # 存档系统
├── assets/
│   ├── images/         # 图块素材
│   │   ├── tiles/      # 地形图块
│   │   ├── player/     # 勇士精灵
│   │   ├── monster/    # 怪物精灵
│   │   └── items/      # 道具图标
│   └── audio/          # 音效资源
└── data/
    └── maps.json       # 楼层数据
```

### 4.4 可选增强方案

| 增强需求 | 技术方案 |
|---------|---------|
| 更好的状态管理 | 引入简单状态管理库或使用发布-订阅模式 |
| 更丰富的UI | 添加React/Vue作为UI层，Canvas专注游戏渲染 |
| 素材管理 | 使用精灵图(Sprite Sheet)减少HTTP请求 |
| 关卡编辑器 | 独立的地图编辑工具 |
